# Task 8.2B-R — 正式移动端复习中心

**优先级：** P0
**依赖：** 8.2A-H、8.2D-B、5.9P-C。
**目标：** 替换复习占位页，并通过服务端 review intent 启动正式 Session。
**非目标：** 不在客户端规划复习，不重写学习算法。

## 开始前

- 读取根 `AGENTS.md`、Post-8.2A 补充设计、相关 ADR 和前置 Task 报告。
- 记录 `git status --short`、diff stat、分支、HEAD 和最近 5 个 commit。
- 不使用 `git reset --hard` 或 `git clean`。
- 不修改无关文件，不自动 push。
- 若前置 Task 未满足，停止并报告，不做临时绕过。

## 页面状态

- loading
- due items
- today complete
- no history
- API error
- cached offline
- offline no cache
- active review session
- session creation failure
- auth expired

## 页面内容

- 到期数、逾期数、预计分钟、next due；
- 汉字、拼音、声调、词句、混淆分组；
- 支持性 reason 文案；
- 离线/同步状态；
- 开始复习/继续复习/刷新。

## 开始复习

必须：

```text
POST session-plan-v2 { intent: review }
```

不得把 Review Center items 传给 Planner 或在客户端排列 Activity。

## 完成刷新

Session 完成和 Outbox 同步后：

- invalidate review-center；
- 更新 due count/next due；
- 不重复创建 Session；
- 离线完成显示 pending sync；
- 同步后自动对齐。

## 测试

完整状态、reason mapping、active review 恢复、离线缓存、跨账号隔离、小屏、动态字体、Screen Reader，以及 E2E：打开复习 → 创建 review session → 完成 → 数量更新。

## 验收

复习 Tab 不再有占位文案，并且完成的是服务器权威的 Review Session。

## 完成报告

必须列出：

1. 修改和新增文件；
2. 合同/算法/迁移版本；
3. 数据兼容与回滚；
4. 实际运行的命令；
5. 每项测试真实结果；
6. 安全、隐私、离线和学习影响；
7. 尚未解决的风险；
8. 下一项唯一建议 Task。

完成后停止，不自动开始下一 Task，不自动 push。
