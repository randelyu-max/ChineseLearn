# Task 8.3E — 生产课程 Release、导入与不可变版本

**优先级：** P0
**依赖：** Session V2、拼音持久化。
**目标：** 新环境不依赖测试 Seed，也能导入并学习正式课程。
**非目标：** 不建设完整 CMS。

## 开始前

- 读取根 `AGENTS.md`、Post-8.2A 补充设计、相关 ADR 和前置 Task 报告。
- 记录 `git status --short`、diff stat、分支、HEAD 和最近 5 个 commit。
- 不使用 `git reset --hard` 或 `git clean`。
- 不修改无关文件，不自动 push。
- 若前置 Task 未满足，停止并报告，不做临时绕过。

## Release Package

建立版本目录、Manifest、媒体清单、审核清单、前版本迁移映射和脚本。

## 发布流程

```text
validate
→ hash
→ import staging
→ DB constraints
→ smoke learn session
→ smoke review session
→ mark published/active
```

已发布版本不可原地更新。Active Version 需要显式指针或唯一约束，不使用“最新发布时间”隐式选择。

## 内容最低目标

公开 V1：100 汉字、200 词、30 句型、10 短故事、基础拼音和声调。若当次 Task 不能完成全部编辑内容，必须生成明确 coverage report，不能把 Demo 当成生产完成。

## 测试

- 空数据库导入；
- 重复导入；
- hash mismatch；
- 未审核内容拒绝；
- 未授权媒体拒绝；
- Active Version；
- Session smoke；
- 旧 Session pinned version；
- rollback runbook。

## 验收

一个全新 PostgreSQL 环境只执行 migration + 正式 import 后，可以登录并创建真实 learn/review Session。
