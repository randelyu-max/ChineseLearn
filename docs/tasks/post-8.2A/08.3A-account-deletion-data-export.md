# Task 8.3A — 账号删除、数据导出与本地清理

**优先级：** P0
**依赖：** 正式数据模型稳定。
**目标：** 提供公开发布所需的完整账号删除和数据可携带能力。
**非目标：** 不实现订阅退款或付费处理。

## API

版本化实现：

- Server data export；
- Account deletion；
- 必要的 deletion status/receipt。

使用最近认证和明确二次确认。不得只提供 Support 邮箱。

## 删除范围

- Better Auth user/account/session/verification；
- profile；
- sessions/activities/attempts/evidence；
- skill/review/confusion；
- signature metadata/summaries；
- 其他用户私有业务数据；
- 本地 SecureStore、SQLite、Outbox、Content Cache、Writing Draft、Web Storage。

旧设备事件必须因无效认证和 user namespace 拒绝，不能恢复已删除数据。

## 导出

Server Export 包含可理解的 JSON/ZIP 和版本说明；不包含内部答案键或其他用户。原始书写轨迹只在设备本地单独导出。

## 外部资源

建立公开网页或表单供用户在 App 外发起删除，并更新 Privacy Notice、Store Metadata 和 Data Safety 资料。

## 测试

- reauth；
- 取消确认；
- 幂等删除；
- cascade；
- session revocation；
- local purge；
- 离线旧 outbox；
- 同邮箱重建为新 userId；
- export 内容和隐私；
- A/B 隔离；
- 失败恢复。

## 验收

用户可以在 App 内找到并完成全账号删除；删除后旧 token 和旧离线数据不能恢复服务器状态。
