# Task 8.2D-B — 通用汉字 Session Runner 与首页正式接入

**优先级：** P0
**依赖：** 8.2D-A。
**目标：** 用通用 Runner 完成四种汉字题的真实学习闭环，并从首页移除 Demo 入口。
**非目标：** 不接六种拼音题，不做复习页面。

## 开始前

- 读取根 `AGENTS.md`、Post-8.2A 补充设计、相关 ADR 和前置 Task 报告。
- 记录 `git status --short`、diff stat、分支、HEAD 和最近 5 个 commit。
- 不使用 `git reset --hard` 或 `git clean`。
- 不修改无关文件，不自动 push。
- 若前置 Task 未满足，停止并报告，不做临时绕过。

## 首页

“继续学习”流程：

1. 恢复现有 Active Session；
2. 没有则创建 `intent=learn`；
3. 无安全内容时显示可操作错误；
4. 进入正式 Session Runner。

生产导航不得指向 `/demo-course` 或 Showcase。

## Runner

支持：

- audio_to_glyph
- glyph_to_image
- word_build
- sentence_order

统一实现：

- Header、进度、预计剩余；
- 音频加载；
- Pinyin Support Presentation；
- 提示和重试；
- 支持性反馈；
- 本地即时评分；
- Attempt 与 Snapshot 原子保存；
- Outbox；
- 下一题；
- Session complete；
- 完成摘要；
- 错误/离线/恢复。

客户端本地评分只用于即时反馈，最终结果以 Attempts V2 响应为准。若服务器结果与本地不一致，记录安全遥测并以服务器为准，不显示内部答案。

## 状态和无障碍

覆盖 loading、offline cached、feedback、persisting、sync pending、complete、fatal content error。动态字体、屏幕阅读器、Reduced Motion、小屏和键盘路径必须可用。

## Demo Route

- 保留为开发工具时加明确 `__DEV__` 或内部入口；
- 生产构建/导航测试禁止访问；
- 商店/发布截图不得使用 Demo；
- 首页和 E2E 不再依赖 Demo fixture。

## 测试

- 四题型完整 Session；
- 多 Lesson；
- 本地与服务器评分一致；
- 错题/重试/提示；
- 离线完成后同步；
- App 重启继续；
- 完成后状态刷新；
- 双击/重复提交；
- 登录过期；
- 生产导航无 Demo；
- E2E：登录 → learn → 完成 → mastery/review 更新。

## 验收

首页正式用户能够完成一场服务器计划、离线可恢复、服务端可验证的汉字 Session。
