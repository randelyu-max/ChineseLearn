# Task 8.3D — 诊断 UI、持久化与学习起点应用

**优先级：** P1
**依赖：** 通用 Runner 和生产诊断内容。
**目标：** 把 `diagnostic-v1` 从纯算法变成可用的 Onboarding 能力。
**非目标：** 不做语音录制或 AI。

## 开始前

- 读取根 `AGENTS.md`、Post-8.2A 补充设计、相关 ADR 和前置 Task 报告。
- 记录 `git status --short`、diff stat、分支、HEAD 和最近 5 个 commit。
- 不使用 `git reset --hard` 或 `git clean`。
- 不修改无关文件，不自动 push。
- 若前置 Task 未满足，停止并报告，不做临时绕过。

## 设计

- 音频优先；
- 5–7 分钟；
- 可暂停、恢复、跳过；
- 无负面标签；
- 固定内容版本/算法版本；
- 结果只作为初始先验。

## 数据

新增 `diagnostic_runs`，强制 RLS。保存严格结果摘要，不保存原始语音。

## 起点应用

Planner 读取最近完成结果：

- 初始课程范围；
- 首周拼音比例；
- pinyin support mode 建议；
- 新概念数量上限。

真实 Attempts 后逐渐覆盖诊断先验。

跳过时使用补充设计默认路径。

## 测试

五类用户 fixture、跳过、暂停恢复、时间/题目上限、结果持久化、跨用户、Planner 起点、后续 Evidence 覆盖、无网络。

## 验收

首次用户完成或跳过诊断后，首页生成的第一场正式 Session 明显使用相应起点。
