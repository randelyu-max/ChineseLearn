# Task 6.5W — 中文姓名书写覆盖与签名范围硬化

**优先级：** P1
**依赖：** 既有 6.1W–6.4W。
**目标：** 让公开 V1 对姓名书写的承诺真实、覆盖可测、隐私边界清楚。
**非目标：** 不做书法评分、真伪验证、第三方签名模仿或 AI。

## 姓名模型

- 原文和确认读音分离；
- 多音字由用户确认；
- Script 偏好不修改姓名；
- Unsupported char 保留。

## 资产构建

建立 writing asset allowlist、generator 和 manifest。目标：

- 前 100 常见姓氏；
- 至少 300 常见名字字；
- 报告 bundle 体积、加载和覆盖；
- 每字来源/许可/hash；
- 无资源时只允许自由书写，绝不猜笔顺。

## UI 文案

统一为“中文名字书写与风格练习”。明确：

- 不是法律签名；
- 不是身份认证；
- 不是专业书法评分；
- 风格预览是确定性几何变化。

## 数据与清理

- 原始轨迹继续 local-only；
- 账号退出/删除时正确隔离和清除；
- Server export 不包含原始轨迹；
- Local export/delete 单独可用。

## 测试

资产覆盖、未知字、多音字、简繁切换、Bundle 性能、轨迹隐私、跨账号、删除清理、无障碍和文案。

## 验收

公开功能对支持范围诚实，大多数常见姓名有标准笔顺，不支持的姓名仍可安全自由书写。
