# Task 8.2C-D — Attempts V2 与规范化 Evidence

**优先级：** P0
**依赖：** 8.2C-C。
**目标：** 移除 Attempts 对单一 Lesson `content_spec` 的依赖，按 Session Activity Snapshot 服务端评分，并规范化多目标 Evidence。
**非目标：** 本 Task 只正式启用四种汉字题；六种拼音在 5.9P-B 启用。

## 合同

新增 `attempts-batch-request-v2`：

- `sessionId`
- bounded attempts
- 每条使用 `sessionActivityId`
- answer union
- occurredAt
- offlineSequence
- responseMs
- hintLevel
- pinyinSupport
- replay/retry count

## 服务端评分

- 查找当前用户 Session Activity；
- 验证 Session active；
- 用 Activity 固定的 `exercise_snapshot` 评分；
- 验证 answer shape 与 exercise type 匹配；
- 不再读取 `learning_sessions.lesson_id`；
- 评分结果、选中值和预期值记录到 immutable Attempt；
- completed/abandoned Session 拒绝新 Attempt。

## attempt_evidence

用下一迁移新增规范化表。对每个 EvidenceTarget 计算：

- correct
- baseQuality
- supportMultiplier
- effectiveQuality
- algorithmVersion

Skill/Review 重放从 Evidence 表读取，并保持稳定排序：

```text
device_event_at
→ offline_sequence
→ attempt_id
→ evidence target key
```

## 实际完成证据（2026-07-24）

### 实现范围

- 新增严格的 `attempts-batch-request-v2` / `attempts-batch-response-v2`；事件使用
  `sessionActivityId`，answer 支持选择和排序联合类型，边界拒绝 user/candidate 等权威字段。
- V2 服务只查当前用户 Session Activity，验证 Session 为 `in_progress`，从固定
  `exercise_snapshot` 重新评分；客户端 `isCorrectClient` 仅作为忽略的诊断元数据。
- 保存不可变的 selected/expected value、离线序号、retry/replay、Pinyin support 与请求
  hash；相同 Attempt ID 的异构重试返回 `ATTEMPT_ID_CONFLICT`。
- 每个 Evidence Target 写一条 `attempt_evidence`，保存 correct、base/support/effective
  quality、axis、concept、skill 与 algorithm version。Skill/Review 按规范化表重放，不再
  扫描 JSONB target 列表。
- V1 分支保持可用，新的 V1 Attempt 同样写规范化 Evidence；历史列没有删除。
- 修正依赖中的高成功率收尾选择并把 materializer 提升到 v2；只有重新计算后预测成功率
  `>= 0.90` 的候选可作为结尾。

### 迁移与回填

- 新增且仅新增 `0010_attempts_v2_normalized_evidence.sql`；`0001`–`0009` 未改写。
- `attempt_evidence` 与 `attempt_batch_v2_events` 都使用 `user_id`、强制 RLS、最窄 own
  policy 和不可变事件语义。
- 历史回填把旧 `evidence_weight` 固定为等价 effective quality，并用
  `ON CONFLICT DO NOTHING` 保证回填段可重放。
- 独立空库测试先应用 `0001`–`0009`、插入两个多目标旧 Attempt，再应用 `0010`：生成
  4 条 Evidence；第二次执行回填段行数不变；mastery、stability、review reason/interval/
  due time 重放与迁移前完全相同。

### 安全、离线、学习影响与回滚

- 所有权只来自认证事务；用户 A 不能读写用户 B 的 Attempt、Evidence 或批次回执。
- Pinyin support 必须与固定 Snapshot 的初始可见/可揭示状态一致，不能通过伪报 `none`
  提高独立汉字 Evidence；客户端正确性不会影响服务端答案。
- 离线事件按 `occurredAt → offlineSequence → attemptId → Evidence target key` 稳定重放，
  批次和单 Attempt 均幂等；移动端持久化 Outbox/缓存仍待 8.2D-A。
- 当前仅评分四种汉字题。六种拼音合同仍可存在于 Snapshot，但 Planner capability gate
  保持关闭，直到 5.9P-B 实现服务端评分。
- 回滚时关闭 Attempts V2 路由分支，保留不可变 Attempt、Evidence 和回执供审计；V1
  兼容分支继续可用。不删除 `0010` 数据、不改写旧迁移。
