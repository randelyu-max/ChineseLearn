# Task 5.9P-B — 六种拼音服务端评分与 Evidence

**优先级：** P0
**依赖：** 5.9P-A、8.2C-D。
**目标：** 正式启用六种拼音 Activity 的 Session 生成、服务端评分、Evidence、Skill State 和 Review。
**非目标：** 不实现拼音移动端正式 Runner。

## 评分映射

逐题型定义版本化评分与 Evidence：

- audio_to_pinyin
- pinyin_to_audio
- pinyin_to_glyph
- glyph_to_pinyin
- tone_choice
- pinyin_syllable_build

每种必须写明：

- answer shape；
- primary/secondary/transfer targets；
- ability axis；
- BKT 参数；
- pinyin support 是否适用；
- 多音字和语境规则；
- tone/neutral tone 规则。

## 拼音提示 Evidence

- 拼音目标本身不因拼音显示而降权；
- 汉字独立识别 Evidence 按 none/visible/revealed/full answer 降权；
- `pinyin_to_glyph` 不得把“有拼音线索答对汉字”记成完整独立认字；
- 所有权重集中在版本化 learning-engine 配置。

## Planner

完成后打开拼音 capability gate。Planner 只发出已有内容、服务端支持、移动端未来可渲染的 Activity。若移动端版本尚不支持，API 需要 App Capability/Min Version 防止旧客户端收到未知类型。

## Review

- tone group；
- due reasons；
- stable dedupe key。
