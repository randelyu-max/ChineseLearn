# Task 8.2D-A — 移动端正式 Session 数据层与恢复

**优先级：** P0
**依赖：** 8.2C-D。
**目标：** 移动端能够获取、缓存、恢复和同步正式 V2 Session，但本 Task 不替换正式页面。
**非目标：** 不实现完整 Runner UI，不接拼音。

## 数据层

建立：

- Session API client；
- `GET active`；
- `POST session-plan-v2`；
- start/complete/abandon；
- V2 Snapshot parser；
- per-user cache；
- sync/invalidation helpers。

## 本地 Schema

升级离线 Schema，要求：

- 按 userId/sessionId/schemaVersion 隔离；
- 保存 V2 Session Header 和 Activity Snapshot；
- Attempt Outbox 引用 sessionActivityId；
- 旧 V1/Demo 数据有明确迁移或隔离；
- 不把 Demo Session 上传为正式 Session；
- 退出后清除或严格隔离当前账号缓存；
- 损坏 Snapshot 单独隔离；
- 导出恢复 JSON 更新版本。

## 恢复流程

```text
Auth ready
→ read local active snapshot
→ query server active when online
→ reconcile by sessionId/schemaVersion/status
→ sync pending attempts
→ render resumable state
```

服务器 completed/abandoned 优先于旧本地 active。不得覆盖未同步 Outbox；先同步或给出明确恢复选择。

## 测试

- 首次创建/缓存；
- 进程死亡恢复；
- 离线恢复；
- 服务器与本地状态冲突；
- completed/abandoned 对齐；
- 两个账号缓存不串；
- V1/Demo migration；
- 损坏 payload；
- schema downgrade 拒绝；
- Outbox 不丢失；
- 全部移动端回归。

## 验收

不依赖 UI，也可以通过测试证明正式 V2 Session 能跨重启、离线和账号切换安全恢复。
