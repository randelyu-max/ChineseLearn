## 请求与结果

实现 `session-plan-request-v2`：

```text
clientSessionId
idempotencyKey
intent: learn | review
targetMinutes
```

结果支持：

- planned
- active_session_exists
- nothing_due
- insufficient_safe_content

不创建空 Session。

## learn Planner

- 复用现有纯学习引擎；
- 保留到期优先、先修约束、0–4 新概念、高难连续限制和成功收尾；
- 从已发布、Active Curriculum Release 选择内容；
- 只选择当前 Attempts V2 能评分、移动端后续能渲染的类型；
- 当前阶段先正式支持四种汉字题；拼音候选必须被 capability gate 排除，不能产生无法提交的 Activity。

## review Planner

- 只使用权威 `review_schedule`、必要混淆项和稳定性检查；
- `due_at <= serverNow`；
- 不引入新概念；
- 不接受客户端 review keys；
- 无到期内容返回 `nothing_due`；
- 保留固定 seed、难度保护和高成功率结尾；
- 同一个 Review Center preview 不等于固定 Exercise，服务端仍选择安全题型。

## Snapshot Materialization

把每个 Planner Candidate 解析为：

- 来源 Lesson/Exercise；
- 完整 LearningExerciseV2；
- content/version/hash；
- Evidence Targets；
- Pinyin Support；
- 经审核幽默或 neutral variant；
- estimatedSeconds。

所有 Activity 和 Session Header 在同一个事务插入。任何 Activity 解析失败时整个 Session 不得半创建。

## Idempotency 与 Active Session

- 相同用户/idempotency key 返回第一次结果；
- 已存在 Active Session 时返回它，不创建新 Session；
- 重试时即使 Profile 或课程版本变化，也返回原 Snapshot；
- 计划创建后不可修改。
