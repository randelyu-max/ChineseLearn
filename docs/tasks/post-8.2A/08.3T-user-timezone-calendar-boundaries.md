# Task 8.3T — 用户时区与日历边界

**优先级：** P1
**依赖：** Profile 和正式统计/复习页面。
**目标：** 正确处理全球用户的“今天、本周、连续学习”和本地到期显示。
**非目标：** 不根据精确位置自动跟踪用户。

## 开始前

- 读取根 `AGENTS.md`、Post-8.2A 补充设计、相关 ADR 和前置 Task 报告。
- 记录 `git status --short`、diff stat、分支、HEAD 和最近 5 个 commit。
- 不使用 `git reset --hard` 或 `git clean`。
- 不修改无关文件，不自动 push。
- 若前置 Task 未满足，停止并报告，不做临时绕过。

## 数据

Profile 增加 IANA `time_zone`。设备检测作为建议，用户可修改。无效值回退 UTC 并提示。

## 时间规则

- 数据库存 UTC；
- due 判断用 UTC instant；
- UI 用用户时区显示；
- 日/周统计按用户时区；
- 时区变化只前向生效，不重写历史日汇总；
- 不收集 GPS；
- 记录 timezone version/changed_at 如统计需要。

## 测试

- UTC± 大时差；
- DST spring/fall；
- 23/25 小时日；
- 跨午夜离线；
- 旅行改时区；
- nextDueAt 显示；
- 周边界；
- 无效时区；
- 固定 clock。

## 验收

海外用户看到的“今天”和周进度不会按服务器时区错误切换。
