## 合同设计

新增版本，不原地改变 V1：

```text
learning-exercise-v2
session-plan-request-v2
session-plan-snapshot-v2
session-activity-v2
```

### SessionIntent

```ts
type SessionIntent = 'learn' | 'review';
```

### SessionActivitySnapshotV2

必须包含：

- `sessionActivityId`：服务端实例 UUID；
- `sourceExerciseId`：课程中的稳定 ID；
- `position`；
- `exerciseType`；
- `contentRef`、`contentVersion`、`contentSha256`；
- `exercise`：严格 `LearningExerciseV2`；
- `evidenceTargets`：严格目标数组；
- `pinyinSupport`；
- `humorContentRef`；
- `estimatedSeconds`。

### LearningExerciseV2

建立正式 discriminated union，覆盖：

现有四种：

- `audio_to_glyph`
- `glyph_to_image`
- `word_build`
- `sentence_order`

预留并完整定义六种拼音 Schema：

- `audio_to_pinyin`
- `pinyin_to_audio`
- `pinyin_to_glyph`
- `glyph_to_pinyin`
- `tone_choice`
- `pinyin_syllable_build`

本 Task 只定义严格合同和 Fixture，不允许 Planner 发出未完成服务端评分的拼音题。

每个 Schema 必须：

- strict；
- 有稳定 Activity/Option/Tile ID；
- 验证正确答案引用合法；
- 验证无歧义；
- 有可访问性文本；
- 有 payload 大小上限；
- 不包含 userId、mastery 或客户端权威字段。

### EvidenceTargetV1

```ts
type EvidenceTargetV1 = {
  conceptType: 'character' | 'word' | 'sentence' | 'pinyin';
  conceptId: string;
  skill: string;
  abilityAxis:
    | 'spoken_audio_comprehension'
    | 'pinyin_recognition'
    | 'tone_discrimination'
    | 'hanzi_recognition'
    | 'word_reading'
    | 'sentence_reading'
    | 'confusion_discrimination';
  role: 'primary' | 'secondary' | 'transfer';
};
```
